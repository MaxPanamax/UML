Создать два вида UML диаграмм: диаграмму классов (Class),
диаграмму прецедентов (UseCase)  для процесса получения денег в банкомате.
Рассмотреть следующие возможные случаи:
1. Проверка пароля, проверка суммы и выдача денег завершены успешно;
2. Проверка пароля выполнена успешно, но запрошенная сумма не может быть выдана по причине недостаточного количества
денег на счету клиента;
*************************************************
диаграмму прецедентов (UseCase) 
*************************************************
@startuml
"Клиент банка" as klient
rectangle банкомат{
klient-->(Вставить карту в банкомат)
(Вставить карту в банкомат)->(Проверка пороля)
(Проверка пороля)->(Проверка суммы выдачи денег)
(Проверка суммы выдачи денег)-down->(Недостаточное колличество денег на счету просьба выбрать другую сумму)
(Недостаточное колличество денег на счету просьба выбрать другую сумму)->(Вставить карту в банкомат)
}
(Проверка суммы выдачи денег)->(Получение денег)
@enduml
****************************************************
диаграмму классов (Class)
****************************************************
@startuml
classКлиент_Банкомата..>classБанкомат
classКлиент_Банкомата : Банковская_карта:bool
classКлиент_Банкомата : Пин-код:int
classКлиент_Банкомата : Сумма:int
classКлиент_Банкомата : Клиент_Банкомата(bool Банковская_карта, int Пин-код,int Сумма)
classКлиент_Банкомата : friend class Банкомат()
note left: В классе Клиент_Банкомата конструктор\n с параметрами и обозначается дружеский класс Банкомат
classБанкомат : Прверка_пароля:bool
classБанкомат : Проверка_наличия_суммы:bool
classБанкомат : Снятие:bool
classБанкомат : Зачисление:bool
classБанкомат : Банкомат()
classБанкомат : Проведение требуемой операции(Клиент банкомата & Клиент, bool Проверка_пороля,bool Проверка_наличия_суммы,bool Снятие,bool Зачмсление)
note left: Метод в классе Банкомат-Проведение требуемой \nоперации прнимает из дружеского класса Клиент_Банкомата\n Банковскую карту, Пин-код,Сумму и Запрашивает свои bool значения\n Проверку_пароля,Проверку_наличия_суммы, \nСнятие,Зачисления и  завершает операцию
@enduml
